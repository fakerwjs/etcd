# Go-Zero 服务注册与发现 - 启动与排查文档

## 一、环境准备

### 1.1 确保 etcd 已启动

```bash
# 检查 etcd 是否运行
curl -s http://127.0.0.1:2379/version

# 如果未启动，启动 etcd
etcd
```

### 1.2 项目结构

```
go-zero-test/
├── user/                    # RPC 服务端（服务提供者）
│   ├── etc/user.yaml        # RPC 服务配置
│   └── user.go              # RPC 服务入口
├── user-api/                # HTTP API 服务（服务消费者）
│   ├── etc/user-api.yaml    # API 服务配置
│   └── user.go              # API 服务入口
```

## 二、启动服务

### 2.1 启动顺序

```
1. 启动 etcd（必须）
2. 启动 user RPC 服务（服务提供者）
3. 启动 user-api 服务（服务消费者）
```

### 2.2 启动命令

**步骤 1：启动 etcd**
```bash
etcd
```

**步骤 2：启动 user RPC 服务**
```bash
cd /Users/dean/work/go-zero-test/user
go run user.go -f etc/user.yaml
```

**步骤 3：启动 user-api 服务**
```bash
cd /Users/dean/work/go-zero-test/user-api
go run user.go -f etc/user-api.yaml
```

### 2.3 启动验证输出

user RPC 服务启动成功：
```
Starting rpc server at 0.0.0.0:8080...
```

user-api 服务启动成功：
```
Starting server at 0.0.0.0:8888...
```

## 三、排查服务是否成功

### 3.1 检查 etcd 是否正常运行

```bash
curl -s http://127.0.0.1:2379/version
```

**成功输出**：
```json
{"etcdserver":"3.6.12","etcdcluster":"3.6.0","storage":"3.6.0"}
```

### 3.2 检查服务是否注册到 etcd

```bash
etcdctl get --prefix ""
```

**成功输出**（服务已注册）：
```
user.rpc/7587895848947838272
192.168.88.238:8080
```

- `user.rpc/xxx` 是注册到 etcd 的 key
- `192.168.88.238:8080` 是服务发现的地址

### 3.3 检查服务端口是否监听

```bash
# 检查 user RPC 端口
lsof -i :8080

# 检查 user-api 端口
lsof -i :8888
```

### 3.4 测试服务调用

**调用 user-api 接口**：
```bash
curl http://localhost:8888/user/123
```

**成功响应**：
```json
{"id":123,"username":"user«","email":"user«@example.com"}
```

## 四、常见问题排查

### 4.1 服务未注册到 etcd

**症状**：调用 API 返回错误或超时

**排查步骤**：
1. 确认 etcd 已启动：`curl -s http://127.0.0.1:2379/version`
2. 查看 RPC 服务日志是否有错误
3. 检查配置文件 `Etcd.Hosts` 和 `Etcd.Key` 是否正确

**配置文件检查**：
```yaml
# user/etc/user.yaml
Name: user.rpc
ListenOn: 0.0.0.0:8080
Etcd:
  Hosts:
  - 127.0.0.1:2379
  Key: user.rpc
```

### 4.2 服务注册了但发现失败

**症状**：RPC 服务已注册，但 API 无法调用

**排查步骤**：
1. 确认 etcd 中的注册 value 是正确的 IP 地址
2. 检查 API 服务的 `Etcd.Hosts` 和 `Etcd.Key` 配置
3. 检查防火墙是否阻止了服务间的通信

**配置文件检查**：
```yaml
# user-api/etc/user-api.yaml
UserRpc:
  Etcd:
    Hosts:
    - 127.0.0.1:2379
    Key: user.rpc
```

### 4.3 端口被占用

**症状**：服务启动失败，提示端口已被占用

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i :8080
lsof -i :8888

# 结束进程或修改配置文件中的端口
```

## 五、配置说明

### 5.1 服务注册配置（RPC 服务端）

文件：`user/etc/user.yaml`

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| Name | 服务名称，用于 etcd 注册 | user.rpc |
| ListenOn | 服务监听地址 | 0.0.0.0:8080 |
| Etcd.Hosts | etcd 服务器地址 | 127.0.0.1:2379 |
| Etcd.Key | 注册到 etcd 的 key | user.rpc |

### 5.2 服务发现配置（API 消费端）

文件：`user-api/etc/user-api.yaml`

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| UserRpc.Etcd.Hosts | etcd 服务器地址 | 127.0.0.1:2379 |
| UserRpc.Etcd.Key | 发现服务的 key | user.rpc |

### 5.3 服务调用流程

```
1. user RPC 启动时向 etcd 注册
   → key: user.rpc/lease_id
   → value: 192.168.x.x:8080

2. user-api 启动时连接 etcd 发现服务
   → 从 etcd 读取 key: user.rpc/*
   → 获取 value: 192.168.x.x:8080

3. user-api 发起 gRPC 调用
   → 连接到 192.168.x.x:8080
```

## 六、快速验证脚本

```bash
# 1. 检查 etcd
curl -s http://127.0.0.1:2379/version && echo "✅ etcd 运行正常"

# 2. 检查服务注册
etcdctl get --prefix "" | grep "user.rpc" && echo "✅ 服务已注册"

# 3. 检查端口监听
lsof -i :8080 | grep LISTEN && echo "✅ user RPC 运行正常"
lsof -i :8888 | grep LISTEN && echo "✅ user-api 运行正常"

# 4. 测试 API 调用
curl -s http://localhost:8888/user/123 && echo "✅ 服务调用成功"
```
