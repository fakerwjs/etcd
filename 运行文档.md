# go-zero 服务注册与发现 运行文档

本项目基于 [go-zero](https://github.com/zeromicro/go-zero)，演示了通过 **etcd** 做服务注册与发现的微服务调用链路。

## 一、架构概览

```
                  ┌──────────────┐
   HTTP 请求 ──▶  │   user-api   │  HTTP 网关，端口 8888
                  └──────┬───────┘
                         │ 通过 etcd 发现 user.rpc，再发起 gRPC 调用
                         ▼
                  ┌──────────────┐
                  │  etcd 注册中心 │  127.0.0.1:2379
                  └──────┬───────┘
                         │ user 服务启动时把自己注册到 key=user.rpc
                         ▼
                  ┌──────────────┐
                  │     user     │  gRPC 服务，端口 8080
                  └──────────────┘
```

三个模块：

| 模块 | 类型 | 监听端口 | 是否用注册发现 | 说明 |
|------|------|---------|--------------|------|
| `user/`     | gRPC 服务 | `0.0.0.0:8080` | ✅ 注册到 etcd（key=`user.rpc`） | 提供 `GetUser` / `Ping` |
| `user-api/` | HTTP 网关 | `0.0.0.0:8888` | ✅ 从 etcd 发现 `user.rpc` | 对外 HTTP，内部转 gRPC |
| `greet/`    | HTTP 服务 | `0.0.0.0:8888` | ❌ 独立服务 | 与注册发现链路无关 |

> ⚠️ `greet` 和 `user-api` 默认都用 8888 端口，**不要同时启动这两个**（演示注册发现只需要 `user` + `user-api`）。

### 关键配置

`user/etc/user.yaml`（服务端：把自己注册进 etcd）：
```yaml
Name: user.rpc
ListenOn: 0.0.0.0:8080
Etcd:
  Hosts:
  - 127.0.0.1:2379
  Key: user.rpc        # 注册的 key
```

`user-api/etc/user-api.yaml`（客户端：从 etcd 按 key 发现服务）：
```yaml
Name: user-api
Host: 0.0.0.0
Port: 8888
UserRpc:
  Etcd:
    Hosts:
    - 127.0.0.1:2379
    Key: user.rpc        # 要发现的 key，需与服务端一致
```

## 二、前置依赖

- Go 1.25.5（项目 `go.mod` 要求）
- etcd（本机已安装：`/opt/homebrew/bin/etcd`）
  - 或使用 Docker

## 三、运行步骤

需要打开 **3 个终端**，分别启动 etcd、user 服务、user-api 网关。注意启动顺序：**etcd → user → user-api**。

### 第 1 步：启动 etcd 注册中心

方式 A — 本机已装 etcd：
```bash
etcd
```

方式 B — 用 Docker：
```bash
docker run -d --name etcd -p 2379:2379 \
  -e ALLOW_NONE_AUTHENTICATION=yes \
  -e ETCD_ADVERTISE_CLIENT_URLS=http://0.0.0.0:2379 \
  bitnami/etcd:latest
```

验证 etcd 可用：
```bash
etcdctl --endpoints=127.0.0.1:2379 endpoint health
```

### 第 2 步：启动 user gRPC 服务（会自动注册到 etcd）

```bash
cd user
go mod tidy        # 首次运行时拉取依赖
go run user.go -f etc/user.yaml
```

看到如下输出表示启动成功：
```
Starting rpc server at 0.0.0.0:8080...
```

此时 user 服务已把自己注册到 etcd 的 `user.rpc` 下，可验证：
```bash
etcdctl --endpoints=127.0.0.1:2379 get user.rpc --prefix
# 输出类似 user.rpc/7587... -> 192.168.x.x:8080
```

### 第 3 步：启动 user-api HTTP 网关（会从 etcd 发现 user 服务）

```bash
cd user-api
go mod tidy        # 首次运行时拉取依赖
go run user.go -f etc/user-api.yaml
```

看到如下输出表示启动成功：
```
Starting server at 0.0.0.0:8888...
```

> 说明：`user-api` 通过 `replace user => ../user` 引用本地 `user` 模块的 gRPC 客户端代码，无需额外配置。

## 四、验证调用链路

### 获取用户信息（HTTP → 服务发现 → gRPC）

```bash
curl http://127.0.0.1:8888/user/1
```

预期返回：
```json
{"id":1,"username":"user1","email":"user1@example.com"}
```

这次请求的完整链路：`curl` → `user-api`(8888) → 通过 etcd 发现 `user.rpc` → gRPC 调用 `user`(8080) → 返回。

### 登录接口（本地逻辑，不走 gRPC）

```bash
curl -X POST http://127.0.0.1:8888/user/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"secret"}'
```

预期返回：
```json
{"token":"mock-jwt-token-for-admin"}
```

## 五、验证服务发现是否真的生效

1. 在 user-api 正常工作时，**直接 Ctrl+C 关闭 user 服务**。
2. 再次 `curl http://127.0.0.1:8888/user/1`，会因为没有可用实例而报错。
3. 重新启动 user 服务，稍等片刻（etcd 重新注册），无需重启 user-api，`curl` 又能成功。

这说明 user-api 是通过 etcd 动态发现 user 实例的，而非写死地址。

## 六、常见问题

| 现象 | 原因 / 解决 |
|------|------------|
| user 启动报连接 2379 失败 | etcd 没启动，或 `Etcd.Hosts` 配置不对 |
| user-api 调用报 `no available instance` | user 服务没启动，或两边 `Key` 不一致（都应为 `user.rpc`） |
| 端口被占用 | `user-api` 与 `greet` 都用 8888，不要同时启动 |
| `go run` 拉依赖慢/失败 | 设置代理：`go env -w GOPROXY=https://goproxy.cn,direct` |

## 七、附：greet 独立服务（与注册发现无关）

```bash
cd greet
go run greet.go -f etc/greet-api.yaml   # 默认监听 8888
```
